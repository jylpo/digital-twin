Архітектура Digital Twin: що куди підключають

1) Сенсори та протоколи

У реальному світі дані приходять із PLC/мікроконтролерів (Embedded), модулів IoT, датчиків. Типові протоколи:

  • MQTT — легкий publish/subscribe для IoT
  • HTTP/REST — прості API для відправки телеметрії
  • OPC UA — промисловий стандарт для автоматизації
  • WebSocket — стрім у веб-додаток/дашборд

2) Data Pipeline (потік даних)

Сирові дані часто “шумні”, тому проходять етапи: валідація → очистка → агрегація → збереження. Далі дані використовують і для онлайн моніторингу, і для навчання моделей (ML).

Валідація Фільтрація шуму Агрегація (1с/10с/1хв) Зберігання Аналітика

3) Модель цифрового двійника

Варіанти моделі (часто комбінують):

  • Physics-based: формули, фізика, параметри (точно, але складно)
  • Data-driven (ML): вчимо модель на історії даних
  • Hybrid: фізика + ML корекція

4) Дашборд і рішення

Інтерфейс показує стан, тренди, аномалії, прогнози, алерти. У нашому демо: Dashboard будує графіки з “телеметрії”, яку генерує Sim.

Спробуй: Симулятор → потім Дашборд.
